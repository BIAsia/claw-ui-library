# Agent Probe / Signal Cabinet / 003 — Mode-Preserving Spec

## Intent
Rebuild the 003 probe as an **interactive poster-object**, not a web page. The source mode is the original case-003 demo: a single authored surface with sparse language, hand-made energy, and pointer-reactive presence.

## Locked mode
**Single main card / poster surface** floating in a quiet field.

If the composition starts reading like a landing page, dashboard, magazine layout, or stacked feature site, it has failed.

## Hard rules
1. **One dominant object only.**
   - The page may have ambient background traces, but the experience is centered on one main card/poster.
   - No stacked content sections above or below it.
   - No “hero + features + footer” logic.

2. **Sparse text only.**
   - One main title cluster.
   - One short support paragraph.
   - A few micro-labels / note fragments / stamped phrases.
   - No long explanatory blocks, multi-column copy, FAQ patterns, or product marketing copy.

3. **Pointer-reactive is mandatory.**
   - The main card must visibly respond to the pointer.
   - Motion should feel like a physical printed object being examined, not generic UI hover animation.
   - Depth shifts, spotlight, tilt, drifting labels, and embedded highlight response are allowed.

4. **Graphic-led, not component-led.**
   - Shapes, marks, strokes, stickers, loops, underlines, and composition carry the experience.
   - Standard UI containers must disappear into the artwork.
   - Interface affordances should feel printed into the poster.

5. **No traditional sections.**
   - Ban: feature grids, benefit rows, metrics bars, testimonials, CTA bands, navigation header shells.
   - Ban: evenly spaced card systems or SaaS framing.
   - Ban: “website” reading order.

## Visual behavior
- Warm paper background with red/pink authored accents.
- Condensed, oversized, uppercase display type.
- Small supporting sans text used as notes only.
- Asymmetry is required.
- Large open negative space is required.
- Slight imperfection is desirable; over-clean spacing is not.

## Signal Cabinet translation
The concept remains **Signal Cabinet**.

That means the poster should imply:
- collected scraps
- pattern emergence
- unstable archive logic
- fragments becoming a system

But this must be expressed through **poster composition**, not through literal app structure.

## Required content pattern
Include only these content layers:
- Main title: Signal Cabinet
- Tiny index / code markers for the probe
- One short statement about fragments/signals/archive logic
- A few embedded labels that behave like cabinet tags, routing marks, or active notes

## Interaction pattern
- Card tilt follows pointer.
- A soft inspection light follows pointer.
- Some internal graphic or label layers may drift at different depths.
- Motion must remain restrained and immediate.
- No gimmicky elastic bounce.

## Rejection criteria
Reject the page if any of the following are true:
- It reads like a normal product website.
- It contains multiple obvious sections.
- It explains too much in text.
- The graphics feel secondary to boxes/components.
- Pointer response is weak or absent.
- The experience could be mistaken for generic AI-generated editorial UI instead of a mode-locked poster object.
