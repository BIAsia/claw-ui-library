# Case 004 — Mode-Preserving Spec

## Purpose
Reframe **Signal Cabinet** in the locked mode of **case-004 / Booookin Shop Grid**.

This page is **not** a landing page, manifesto page, or product-marketing homepage.
It must behave like a **browse-first catalog surface** where entries are scanned as objects before they are read as stories.

---

## Locked Mode
**Structured catalog browser**

Signal Cabinet must present fragments as:
- collectible entries
- inventory-like objects
- shelf/grid material
- selectable covers with light metadata

The page should feel like a shop/archive/catalog first.
If it starts feeling like a normal “beautiful website”, it has already drifted.

---

## Non-Negotiables
1. **Browse-first grid is mandatory.**
   The main experience must open directly on a dense catalog/shelf grid.

2. **Object-led covers carry the page.**
   Every entry needs a strong cover/object tile. Covers must do most of the visual work.

3. **Metadata stays light.**
   Labels should be short, quiet, and inventory-like: type, shelf, date, status, source, duration.

4. **No marketing sections.**
   Do not add hero storytelling bands, feature explanations, “why this matters”, benefit blocks, testimonial logic, oversized stats, or conversion copy.

5. **No generic web hierarchy.**
   Avoid a page structure like: hero → explanation → feature cards → CTA.

6. **Catalog logic over narrative logic.**
   Grouping, filters, sort, shelf labels, and counts are valid. Long paragraphs are not.

7. **Flat retail/editorial rhythm.**
   Repetition and scanability should dominate over dramatic composition.

---

## Visual Rules
- Warm paper background, restrained ink, low-chroma accents.
- Thin dividers, clear module boundaries, high scan speed.
- Typography should feel editorial-retail, not startup-product.
- Covers should vary enough to feel collectible, but remain within one disciplined system.
- The eye should move across the grid horizontally/vertically like browsing a shelf.

---

## Component Rules
### Required
- top inventory header / masthead strip
- compact catalog controls (filter / sort / search / format)
- results count or shelf count
- dense multi-column grid
- item cards with:
  - cover
  - title
  - one or two metadata lines max

### Allowed
- quiet status chips
- compact toggle controls
- small section dividers
- shelf headings
- inventory labels

### Forbidden
- explainer sidebars
- philosophy copy blocks
- assistant behavior sections
- big narrative intros
- oversized featured-article treatment
- dashboard metrics as visual centerpiece
- nested card-inside-card marketing compositions

---

## Content Model for Signal Cabinet
Entries should read like captured fragments, such as:
- scan
- note
- image
- tape
- transcript
- clipping
- field record

Useful metadata:
- source type
- issue number
- shelf code
- date
- duration
- status

Avoid long descriptions. If explanation is needed, compress it into one compact line.

---

## Acceptance Heuristic
The page passes only if:
- the first impression is **catalog/browser**, not **site/landing page**
- most of the viewport is occupied by browsable entries
- covers dominate over prose
- explanatory copy is minimal
- a user can scan the collection without reading paragraphs

If someone says “this looks like a nice website for Signal Cabinet”, that is too soft.
If someone says “this feels like a shelf/catalog of Signal Cabinet fragments”, the mode is correct.
