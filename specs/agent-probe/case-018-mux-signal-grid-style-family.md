# Agent Probe / Case 018 / Mux Signal Grid — Style-Family Spec

## Product role decision
**Case 018 should become a live media monitoring wall: a signal-state overview for streaming channels, broadcast feeds, or multiplexed media pathways where a grid of active units expresses presence, status, and change rather than a conventional analytics dashboard.**

This is a signal matrix.

---

## Why this mode fits the source
- The title combines **Mux**, **signal**, and **grid**, strongly implying multiplexed channel logic.
- The animated GIF reference suggests **live state or flicker behavior**, not static poster composition.
- Signal grids are about **simultaneous channel awareness**, which fits repeated cells well.
- This mode prioritizes **status, continuity, and interruption cues** over narrative copy.
- The source likely feels infrastructural and media-technical rather than business-analytical.

---

## Style-family thesis
This family is for:
- broadcast routing walls
- stream health overviews
- multi-feed monitoring
- media ops surfaces
- capture channel review boards
- signal-presence interfaces

It is good at:
- simultaneous visibility
- status scanning
- channel comparison
- technical atmosphere

It is bad for:
- deep numerical analysis
- content archives
- campaign marketing
- long explanations

---

## Non-negotiable visual locks
1. **A repeated signal grid must dominate the interface.** Cells should feel like active channels or signal windows.
2. **Motion or state change should matter.** Blinking, shifting, or updating behavior must communicate live-ness, not gimmick.
3. **Status coding must remain legible at a glance.** Healthy, degraded, idle, or interrupted states should be visually distinct.
4. **Text should stay secondary to signal presence.** Labels and identifiers are useful, but the grid behavior is the primary language.
5. **The mood must stay infrastructural.** Do not soften it into a consumer media gallery.

---

## Concept probe suggestion
**Relay Matrix** — a streaming control wall that shows sixteen live channels, packet health, recording state, and silent failures through an animated signal grid.

---

## Acceptance checklist
- The page reads as a **live media or signal overview**.
- Grid cells feel like channels, not decorative tiles.
- Motion conveys real status information.
- Text labels support but do not dominate the interface.
- The atmosphere feels technical and live.
- The concept plausibly belongs in a broadcast or streaming environment.

---

## Failure tests
- **Fail if** the grid behaves like a generic dashboard card layout.
- **Fail if** animation is purely decorative.
- **Fail if** the page feels like a media gallery instead of a monitoring surface.

- **Pass if** a reviewer can imagine using it to watch many signals at once.
- **Pass if** state changes are visible before details are read.
- **Pass if** the whole experience feels like an engineered signal matrix.
