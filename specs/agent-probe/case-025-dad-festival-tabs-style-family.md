# Agent Probe / Case 025 / D&AD Festival Tabs — Style-Family Spec

## Product role decision
**Case 025 should become a programme-navigation festival hub: a tab-driven event surface for talks, workshops, screenings, awards, or venues where sectional tabs are the primary browsing mechanic rather than a long scrolling page.**

This is a tabbed programme index.

---

## Why this mode fits the source
- The title explicitly references **festival tabs**, making sectional navigation the key design decision.
- Festival content naturally splits into **tracks, dates, venues, formats, and speakers**.
- Tabs work well when users need **fast switching between related schedules** without losing context.
- “D&AD” suggests a design-forward public programme where organization and identity both matter.
- The source likely balances **bold category labels with structured content panes**.

---

## Style-family thesis
This family is ideal for:
- festival sites
- conference schedules
- summit programme hubs
- multi-track event navigation
- award-week planning pages

It is good at:
- structured browsing
- category switching
- reducing scroll fatigue
- preserving context across related sections

It is bad for:
- one-sheet posters
- dense archival grids
- technical monitoring tools
- document-heavy reading experiences

---

## Non-negotiable visual locks
1. **Tabs must be visually and functionally central.** They are the product grammar, not a secondary widget.
2. **Each tab should reveal a distinct but structurally related programme slice.** Users should understand the taxonomy instantly.
3. **The page should feel like a live event hub.** Dates, venues, tracks, or speaker lists should feel actionable.
4. **Visual identity must survive tab switching.** The interface cannot become plain component-library tabs.
5. **Content inside each tab should stay structured and scan-friendly.** Think schedules, lineups, and concise descriptions.

---

## Concept probe suggestion
**Signals Festival Guide** — a design-and-code festival hub with tabs for Talks, Workshops, Installations, City Map, and Tickets.

---

## Acceptance checklist
- The page reads as a **festival or programme hub**.
- Tabs are the unmistakable navigation centerpiece.
- Category structure is clear and useful.
- Each tab pane contains organized public-programme content.
- Switching sections feels coherent and branded.
- The concept plausibly supports a real multi-track event.

---

## Failure tests
- **Fail if** the tabs could be removed without changing the page meaningfully.
- **Fail if** each tab feels like random unrelated content.
- **Fail if** the interface becomes generic application settings UI.

- **Pass if** the tabs clearly organize a complex public programme.
- **Pass if** users can move between event slices without losing orientation.
- **Pass if** the overall experience feels festival-specific rather than generic web navigation.
