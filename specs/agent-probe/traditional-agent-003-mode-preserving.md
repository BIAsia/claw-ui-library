# Agent Probe / Traditional Agent / 003 — Mode-Preserving Spec

## Core intent
Rebuild probe **003** as the **marketing / entry expression layer** for the Traditional Agent concept.

This page is the first emotional contact with the product. It should make Traditional Agent feel:
- dependable
- procedural
- human-facing
- task-oriented
- slightly severe in a good way

It is **not**:
- the actual app workspace
- a queue management screen
- a dashboard
- a task list
- a documentation page
- a feature comparison page

It is a **front-door operator poster**: a single authored sheet that compresses the promise of a classical AI agent into sparse commands, routing language, and one memorable object-like surface.

The source mode to preserve is the original `case-003-julie-doucet` demo: one dominant poster/card, oversized condensed type, distributed title events, small note fragments, cut-paper accents, scribble structure, and restrained pointer-reactive inspection.

---

## Product role
Traditional Agent 003 should behave like:
- a launch poster
- an operator manifesto cover
- an intake surface
- an entry-state artifact before the product becomes functional

It should communicate only:
1. the concept name (`Traditional Agent`)
2. the product’s procedural promise in very short fragments
3. a few route markers that suggest the rest of the system

The user should feel:
- “this thing receives work and moves it forward”
- not “I have already entered the full product”

---

## Locked mode
**One poster-object only.**

The page must read as a single printed object floating in a dark field.
If it starts reading like a website layout with sections, it has failed.

Allowed outer framing:
- dark quiet surround
- centered poster object
- subtle spotlight / inspection glow
- light physical depth response

Forbidden:
- nav bar
- feature grid
- multiple page sections
- footer
- side panel
- large CTA strip
- “how it works” stack
- app preview screenshots beside the poster

---

## Role translation for Traditional Agent
The page should translate the product into the smallest possible set of operator verbs.

Preferred semantic territory:
- intake
- route
- queue
- run
- handoff
- review
- report
- archive
- escalate
- publish

The important thing is **procedural tone without explanation density**.
It should feel like a machine room announced through poster language.

---

## Composition lock: anchor to the original demo
The layout must remain visibly anchored to the original `case-003-julie-doucet` logic.

Preserve:
- single tall poster ratio
- multiple oversized title events distributed down the sheet
- asymmetrical left/right alternation
- relationship between huge red condensed words and tiny notes
- pink collage shapes as structural support
- hand-drawn loops / zigzags / marks as connective tissue

Do **not**:
- flatten the whole thing into one neat centered title
- replace the staggered title islands with a conventional hero heading + subheading
- turn the poster into a clean corporate editorial cover
- rebalance everything into symmetry

This is still a strange authored poster surface — just translated into Traditional Agent language.

---

## Content budget
Content must remain sparse.

### Allowed
- **1 main concept label**: `Traditional Agent`
- **1 short support statement**: about 8–16 words total
- **1 small probe/index marker**
- **3–6 keyword tags or route fragments**
- **0–3 micro notes**

### Forbidden
- long paragraphs
- feature descriptions
- use-case explanation blocks
- trust/ROI copy
- pricing / CTA language
- FAQ-style explanation

If the page starts teaching, it has failed.

---

## Title/event placement constraints
Large condensed words must behave like fixed poster events.

Suggested event vocabulary:
- TRADITIONAL
- AGENT
- INTAKE
- ROUTING
- QUEUE
- HANDOFF
- RUNBOOK
- REVIEW
- REPORT
- RELEASE

Preferred spatial logic:
- top-left event
- upper-right event
- mid-left event
- mid-right event
- lower-left event
- bottom broad event

The exact words may change, but they must stay distributed like the original demo rather than stacked into a normal heading hierarchy.

Forbidden title behavior:
- one giant centered `Traditional Agent`
- evenly spaced rows of labels
- a normal marketing hero structure

---

## Graphic system
Preserve the original material language:
- warm paper field
- bold red display type
- light pink cut-paper forms
- hand-drawn loops / braces / strokes
- visible negative space
- authored irregular rhythm

For this concept, the graphics should subtly imply:
- intake slips
- routing markers
- stamps
- queue tags
- dispatch stickers
- review annotations

But these remain **graphic supports**, not literal UI widgets.

---

## Interaction behavior
Pointer response is required and must stay restrained.

Required:
- whole poster tilts with pointer
- inspection light follows pointer
- internal layers drift slightly by depth

Forbidden:
- bouncy motion
- flashy hover gimmicks
- interaction that changes layout structure
- motion that makes the page feel like an app screen

The correct feeling is:
**inspecting a printed operator poster**
not
**playing with a landing page toy**

---

## Product messaging tone
The tone should feel procedural and classical.
Short language examples:
- intake first, reasoning later
- route the brief
- queue before drift
- assign / run / review
- every task gets a record
- handoff with trace

Avoid language that sounds mystical, cultural, or archive-poetic in the Signal Cabinet sense.
Traditional Agent should feel more practical, less atmospheric.

---

## Acceptance checklist
A correct implementation should satisfy all of these:
1. Reads immediately as a **single poster-object**
2. Feels like a **marketing / entrance layer**, not the actual product screen
3. Keeps the original 003 demo’s **distributed title-event composition**
4. Uses **very little text**
5. Feels recognizably like **Traditional Agent**, not Signal Cabinet with renamed copy
6. Pointer response feels like **physical inspection**
7. Does not read like a generic AI startup hero

---

## Rejection criteria
Reject if any are true:
- it reads like a homepage
- it reads like a task dashboard
- it explains the product with normal sections
- the large-word placement has been reorganized into a conventional hero
- the page loses the red/pink/paper authored poster mood
- pointer interaction is weak or absent
- the concept still feels poetic/archive-first instead of classical/procedural
