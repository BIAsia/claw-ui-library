# UI Kit / 004 — Mode-Preserving Spec

## Product role
004 is the **resource management / browsing layer** for UI Kit.

It is not:
- a campaign page
- docs essay page
- analytics dashboard
- release note page

It is:
- the place where kit assets are stored and scanned
- the shelf where components, tokens, patterns, and templates can be browsed as holdings
- a catalog-first experience where the user decides what to inspect next

## Locked mode
**Preserve the Booookin browse-grid demo mode.**

The page must still read as:
- slim masthead
- compact utility rows
- dense repeated equal-width columns
- cover/object-led browsing
- thin inventory metadata around each object
- strong divider rhythm

No hero preamble. The grid is the product.

## UI Kit translation
Objects should behave like archived holdings such as:
- component sheet
- token swatch set
- pattern study
- layout template
- motion timing card
- rail/chrome module

Each tile should read first as a **visual object**, then as metadata.

### Metadata budget
Prefer one short top line + one short bottom cluster:
- type
- issue
- shelf
- date
- source
- format
- optional duration/status

No paragraphs.

## Required interface pieces
- compact cabinet masthead
- one or two slim control strips max
- counts / shelf scope indicators
- multi-column repeated grid
- cover-led asset cards
- light search/filter/sort tools

## Forbidden drift
Reject if it becomes:
- a generic component docs homepage
- a modern dashboard with large cards
- a sidebar-heavy design system app
- a portfolio gallery with long captions

## Acceptance
Pass only if:
1. first read is **asset catalog**, not marketing
2. most of the viewport is occupied by browsable objects
3. covers carry more weight than text blocks
4. metadata stays light and archival
5. the Booookin grid logic is still legible
6. the surface plausibly supports browsing a UI Kit collection
