# UI Kit / 005 — Mode-Preserving Spec

## Product role
005 is the **editorial / publication layer** for UI Kit.

It is not:
- the browser/catalog layer
- token management UI
- release dashboard
- docs left-nav shell

It is:
- the publication desk for releases, essays, launch framing, and editorial notes
- the place where the kit is narrated and contextualized
- a magazine-like issue surface that gives authored meaning to system updates

## Locked mode
**Preserve the Self Esteem editorial demo mode.**

The page must still feel like:
- three-column editorial framing
- sparse left rail navigation
- one large center feature story
- a right-side issue list / note stack
- quiet tactile scenic image/object field
- publication rhythm before utility chrome

## UI Kit translation
This route should publish things like:
- release issues
- launch letters
- assembly principles
- design system essays
- notes on why certain components/tokens exist

The editorial voice can be product-facing, but should not collapse into documentation.

## Content structure
Recommended main feature:
- publication title: `UI Kit`
- current issue marker / date
- one feature headline about shipping or assembling the kit
- short dek about components/tokens becoming a living product
- right rail containing two short note/release entries

## Scenic image guidance
The image area should still behave like an editorial photograph/scene surrogate, but may be translated into:
- stacked boards
- pinned sheets
- quiet desk objects
- release material artifacts

It should not become a literal app screenshot.

## Acceptance
Pass only if:
1. the page reads as **editorial/publication**, not management UI
2. the original 005 magazine-like composition remains visible
3. the image field stays atmospheric and restrained
4. UI Kit feels like the subject of a publication, not just a dashboard title
5. the route forms a clear 005 role distinct from 003 and 004
