# Agent Probe / Case 024 / Radiiio Station Table — Style-Family Spec

## Product role decision
**Case 024 should become a broadcast schedule table: a station-centric playlist and programming board for radio, stream channels, or audio residencies where tabular structure is the hero rather than a media gallery.**

This is an on-air table system.

---

## Why this mode fits the source
- The title joins **station** and **table**, which strongly implies ordered broadcast listings.
- “Radiiio” points toward audio culture, channel identity, and programming rhythm.
- Tables are ideal when users need to compare **show title, host, start time, duration, and format**.
- The likely composition values **row/column discipline** more than rich imagery.
- This mode is useful for live radio or stream products where the schedule is itself a major content surface.

---

## Style-family thesis
This family is suited to:
- station schedules
- playlist boards
- host rosters
- residency calendars
- broadcast timetables
- audio programming indexes

It is good at:
- comparison
- sequence clarity
- operational usefulness
- public schedule reading

It is bad for:
- image-heavy promotion
- exploratory archives
- persuasion-led campaigns
- dense analytics

---

## Non-negotiable visual locks
1. **A true table structure must define the experience.** Rows and columns should carry the product logic directly.
2. **Station and program identifiers must be easy to scan.** Show name, host, slot, and status should align predictably.
3. **The page must feel on-air and alive, not bureaucratic.** Small status cues, current-slot highlights, or subtle rhythm can help.
4. **Imagery must remain secondary to the schedule structure.** Do not turn the table into a card gallery.
5. **The tone should preserve audio-culture character.** It must feel like a station system, not an office spreadsheet.

---

## Concept probe suggestion
**Night Frequency Table** — a web radio programming board listing live shows, rebroadcasts, hosts, and genre bands across a 24-hour cycle.

---

## Acceptance checklist
- The first read is **broadcast schedule table**.
- Table columns are doing real semantic work.
- Current/live status is easy to identify.
- The page feels designed, not merely tabulated.
- Audio-culture identity remains visible.
- The concept plausibly supports a radio or streaming station.

---

## Failure tests
- **Fail if** the table collapses into a series of unrelated cards.
- **Fail if** imagery overtakes schedule readability.
- **Fail if** the result feels like office admin UI.

- **Pass if** a user could actually use the page to follow station programming.
- **Pass if** the table itself carries the visual identity.
- **Pass if** the atmosphere still feels cultural and on-air.
