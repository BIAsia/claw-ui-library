# Agent Probe / Signal Cabinet 005 — Mode-Preserving Spec

## Purpose
This spec locks **Signal Cabinet / 005** to the compositional mode of the original `case-005-self-esteem-editorial` demo.

The page is **not** a generic product landing page, not a dashboard, and not a feature-marketing stack.
It must behave like a **magazine front page** for the Signal Cabinet concept.

## Locked mode
**Magazine-style editorial front page**

Signal Cabinet may change the subject matter, copy, and publication concept.
It may **not** change the page into a productized SaaS layout.

---

## Structural model (mandatory)
The page must preserve this three-part front-page structure:

1. **Left rail / publication navigation**
   - vertical, sparse, publication-led navigation
   - reads like issue/site sections, not app IA
   - includes a publication signature or subscription/archive style affordance

2. **Center column / masthead + hero + lead story**
   - oversized masthead/title relationship
   - date / issue metadata at header level
   - one dominant scenic visual field
   - one lead editorial headline + short byline/meta + short deck
   - one issue marker / “See more” style continuation cue

3. **Right rail / issue blocks**
   - a small number of article-like supporting entries
   - number-led or issue-led blocks
   - compact, editorial, quiet
   - not cards for product features

---

## Non-negotiables

### 1) Magazine front page, not product page
The page must read first as a **publication cover/homepage**.
If a viewer can mistake it for a startup landing page, the mode has failed.

### 2) Masthead/hero relationship is sacred
The main title and scenic field must establish the core mood immediately.
No generic intro banners, utility bars, search-first headers, or app-shell framing should compete with this.

### 3) Scenic field is required
A single atmospheric visual field is mandatory.
It should feel like an editorial image plate or photographic scene, not an illustration slot for a feature card.

### 4) Issue blocks, not feature sections
Supporting content must appear as **issue/article blocks** with numbering, date/label, title, and short editorial copy.
Do not convert the right rail into benefits, modules, product pillars, or explainer cards.

### 5) Publication-led navigation only
Navigation labels must feel like they belong to a publication or issue system:
- Issue
- Archive
- Notes
- Letters
- Subscribe
- Desk
- Dispatches

Avoid labels that feel like app/product chrome:
- Features
- Pricing
- Solutions
- Integrations
- Dashboard
- Analytics
- Settings
- Get started

---

## Composition rules

- Use a **three-column editorial shell** on desktop.
- Keep the center column dominant.
- Keep the right rail narrower and quieter than the hero column.
- The hero image should sit in the upper-middle body of the page.
- The lead headline must sit directly below the scenic field.
- The bottom of the lead area should close with an issue marker and continuation cue.
- Whitespace must create pacing; do not fill gaps with extra components.

---

## Typography rules

- Strong masthead with compressed editorial weight.
- Supporting contrast may use a serif italic accent for cultural/editorial flavor.
- Dates, labels, bylines, and reading-time style metadata are allowed.
- Type hierarchy must feel like a magazine issue system, not a product marketing page.

---

## Allowed content translation for Signal Cabinet
Signal Cabinet can reinterpret the content as:
- an issue of collected signals
- fragments before classification
- field notes / archive notes / dispatches
- cabinet entries / editorial observations

But this translation must remain inside the magazine mode.

---

## Explicit prohibitions
Do **not** add or emphasize any of the following:

- generic product hero copy
- feature grids
- benefits sections
- testimonial rows
- KPI/metrics bands
- CTA-heavy conversion blocks
- dashboard widgets
- settings-like controls
- tabbed product modules
- “how it works” sections
- card-grid marketing stacks

If the page starts to read like a startup site with editorial styling on top, it is wrong.

---

## Acceptance test
A passing page should satisfy all of the following:

- It is immediately legible as a **magazine front page**.
- The **masthead + scenic field + lead story** dominate first impression.
- Supporting content reads as **issue blocks/articles**, not product features.
- Navigation reads as **publication-led**.
- The overall effect stays close to the original `case-005` demo’s editorial homepage mode.
- The concept is still recognizably **Signal Cabinet**.
